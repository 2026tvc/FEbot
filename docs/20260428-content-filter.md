# コンテンツフィルター機能追加メモ

## Intent

基本情報技術者試験に関係のない質問（天気、料理、雑談など）がSlackに入力された際に、IT・プログラミング関連の質問のみを受け付けるようにフィルタリング機能を追加する。

## Change

### 新規ファイル

- `src/febot/content_filter.py`
  - LLM（gpt-4o-mini）を使用して質問の関連性を判定
  - IT・プログラミング関連の質問のみを許可
  - 無関係な質問は「申し訳ございませんが、その質問は基本情報技術者試験やIT・プログラミングに関連していないため、回答できません。」と返答

### 変更ファイル

- `src/febot/config.py`
  - `Settings` dataclass に `content_filter_enabled: bool` フィールドを追加
  - 環境変数 `CONTENT_FILTER_ENABLED` からの読み込みを実装（デフォルト: `true`）

- `src/febot/slack_app.py`
  - `ContentFilter` クラスのインポートを追加
  - `create_app()` 関数内で `ContentFilter` インスタンスを生成
  - メンション（`@app_mention`）とDM（`message.im`）の両方でフィルタリング処理を追加
  - RAG処理の直前（199行目付近）にフィルタリングチェックを挿入

- `.env.example`
  - `CONTENT_FILTER_ENABLED` 設定項目を追加（コメントアウト形式で記載）

- `README.md`
  - 機能要件にコンテンツフィルター機能を追加
  - セットアップ手順の環境変数説明に `CONTENT_FILTER_ENABLED` を追加

- `docs/20260428-content-filter.md`（本ファイル）
  - 変更履歴を記録

## 実装詳細

### フィルタリングロジック

1. ユーザーからの質問を受信
2. クイズ判定（`_wants_quiz()`）が `False` の場合
3. RAG が有効で、かつ `content_filter` が有効な場合
4. `ContentFilter.validate()` で質問を判定
   - LLM に以下の基準でOK/NGを判定させる：
     - **許可**: 基本情報技術者試験、プログラミング、IT技術、コンピュータサイエンス
     - **拒否**: 上記以外（天気、料理、観光、雑談など）
5. NG の場合は拒否メッセージを返し、処理を終了
6. OK の場合は通常の RAG 処理を継続

### フィルタリング判定プロンプト

```
あなたは質問の内容を判定するフィルターです。
以下の基準で質問が適切かどうかを判定してください：

【許可する質問】
- 基本情報技術者試験に関する質問
- プログラミング全般に関する質問（Python, Java, C言語など）
- IT技術全般に関する質問（データベース、ネットワーク、セキュリティ、アルゴリズムなど）
- コンピュータサイエンスに関する質問

【拒否する質問】
- 上記以外の質問（天気、料理、観光、雑談など、IT・プログラミングに無関係な内容）

質問を判定し、以下のいずれかのみを返してください：
- 許可する場合: "OK"
- 拒否する場合: "NG"
```

### エラーハンドリング

- フィルター判定中にエラーが発生した場合は、Fail-open（質問を許可）する
- エラーはログに記録され、デバッグ可能

### 設定の無効化

- `.env` で `CONTENT_FILTER_ENABLED=false` を設定することで、フィルタリングを無効化可能
- デフォルトは `true`（有効）

## Impact

### ユーザー体験

- IT・プログラミング関連以外の質問には明確な拒否メッセージを返す
- 無関係な質問に対するRAG処理コストを削減
- ユーザーがボットの対応範囲を理解しやすくなる

### システム

- RAG処理の前にフィルタリングするため、以下のコストを削減：
  - 埋め込みAPI呼び出し（`embeddings.create()`）
  - ベクトル検索（Chroma/Supabase）
  - チャット補完API呼び出し（`chat.completions.create()`）
  - Web検索フォールバック
- フィルタリング自体のコスト：
  - チャット補完API呼び出し1回（最大10トークン）
  - 通常のRAG処理より大幅に軽量

### コード

- `content_filter.py` は独立したモジュールとして実装
- 既存のRAG処理ロジックは変更なし
- テスタビリティを維持

## 使用例

### OK（許可される質問）

- 「基本情報技術者試験の勉強方法を教えて」
- 「Pythonのリスト内包表記について教えて」
- 「データベースの正規化とは何ですか？」
- 「アルゴリズムの計算量について説明して」

### NG（拒否される質問）

- 「今日の天気は？」
- 「おすすめのラーメン屋を教えて」
- 「東京の観光スポットは？」
- 「今日のランチは何がいい？」
