# SKILL同期ポリシー導入

## 変更内容

- ルートに `SKILL.md` を新規追加
- `AGENT.md` の必読ファイルに `SKILL.md` を追加

## 意図（なぜ必要か）

- AI 主導開発で、実装変更に対して `README.md` と `.env.example` が追随せず乖離する問題を抑止するため。
- 人間レビューの前段で、最低限の同期条件を機械的に満たす運用を固定するため。

## 選択理由

- ルールを散在させず、`SKILL.md` に集約することで参照コストを下げる（KISS）。
- 変更トリガーごとに更新対象ファイルを固定し、判断の揺れを減らす（DRY）。
- 過剰な品質ゲートを増やさず、実務で守れる最小セットに限定する（YAGNI）。

## 期待効果

- コミット時点で `README.md` / `.env.example` / 実装の整合性が高まる。
- 環境変数や挙動変更時の説明漏れが減る。
- 新規参加者が運用ルールを1ファイルで把握できる。
