## 問（OS）

次のうち、デッドロックの発生条件として**不適切**なものはどれか。

**ア** 相互排除（ある資源は同時に1プロセスだけが使える）  
**イ** 占有と待ち（資源を持ったまま別の資源を待つ）  
**ウ** プリエンプション（実行中のプロセスを強制奪取できる）  
**エ** 循環待ち（待ちの鎖が輪になる）

**正解**: ウ  
**解説**: 古典的な4条件は「相互排除・占有と待ち・非奪取・循環待ち」である。**非奪取**（奪取できない）が条件のひとつで、プリエンプション可能だとデッドロックが起きにくい。設問の「プリエンプション」は条件としては逆方向の要素なので不適切。

## 問（ネットワーク）

TCP と UDP の説明として**最も適切**なものはどれか。

**ア** UDP は常に TCP より遅い  
**イ** TCP は到達確認と再送により信頼性を高める  
**ウ** UDP は必ず順序保証を行う  
**エ** どちらも必ず暗号化される

**正解**: イ  
**解説**: TCP はコネクション型で再送・順序保証などを提供する。UDP はコネクションレスで信頼性はアプリ任せ。速度は一概に比較できず、暗号化は別層（TLS 等）の話。

## 問（データベース）

リレーショナルデータベースの正規化に関する記述として**適切**なものはどれか。

**ア** 正規化を進めるほど必ずクエリが高速になる  
**イ** 第1正規形は「繰り返し属性を排除する」考え方に基づく  
**ウ** 正規化は非正規化と同じ意味である  
**エ** 主キーは正規化とは無関係である

**正解**: イ  
**解説**: 正規化は冗長性や更新異常を減らす。過度な正規化で結合が増え遅くなる場合もある。主キーや関数従属の理解が前提。

## 問（擬似コード）

次の関数 `f(n)` は非負整数 `n` に対して何を返すか。

```
f(n):
  if n <= 1:
    return n
  return f(n-1) + f(n-2)
```

**ア** \(n\) の階乗  
**イ** フィボナッチ数列の第 \(n\) 項（\(f(0)=0, f(1)=1\) から始まる定義に対応）  
**ウ** \(2^n\)  
**エ** \(n^2\)

**正解**: イ  
**解説**: 典型的なフィボナッチの再帰定義。実装ではメモ化や反復に直して計算量を下げるのが実務でも試験でも重要。

## 問（表計算）

セル `A1=2`, `A2=3`, `A3=4` とする。セル `B1` に `=SUM($A$1:A1)` を入力し、`B1` から `B3` まで下方向にコピーしたとき、`B3` の値はどれか。

**ア** 2  
**イ** 5  
**ウ** 9  
**エ** 14

**正解**: ウ  
**解説**: `$A$1` は行・列とも固定、`A1` は相対参照。`B1`=SUM($A$1:A1)=2、`B2`=SUM($A$1:A2)=5、`B3`=SUM($A$1:A3)=9。
